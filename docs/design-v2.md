## 2. 辩论引擎设计

### 2.1 辩论流程

```
用户提问: "景泰蓝的工艺为什么面临失传？"
    │
    ▼
[Dispatcher 分析] → 识别需要 Craft + Heritage 视角，触发辩论模式
    │
    ▼
┌─ Round 1 ─────────────────────────────────────┐
│ 🎨 Craft Expert (主答):                        │
│   "景泰蓝需经过制胎、掐丝、点蓝、烧蓝、磨光、  │
│    镀金六道工序，掐丝和点蓝最考验功夫..."       │
├─ Round 2 ─────────────────────────────────────┤
│ 🏛️ Heritage Expert (质疑/补充):                │
│   "我补充一点——全国掌握完整景泰蓝工艺的传承人   │
│    不到50位，且平均年龄超过60岁。从传承角度看..."│
├─ Round 3 ─────────────────────────────────────┤
│ 📜 History Expert (历史视角):                   │
│   "景泰蓝在明清是宫廷御用，民间无法接触。这种   │
│    历史惯性导致今天公众认知度低，市场小..."      │
├─ Round 4 (可选) ───────────────────────────────┤
│ 🎨 Craft Expert (回应):                        │
│   "正是如此。而且材料成本高——铜胎+珐琅釉料+     │
│    黄金镀层，一副作品成本上万，也阻碍了传承..."  │
└───────────────────────────────────────────────┘
    │
    ▼
[Fusion] 综合三轮辩论 → 结构化输出
```

### 2.2 辩论触发策略

| 触发条件 | 辩论模式 | 参与 Agent |
|---------|---------|-----------|
| 问题涉及单个技艺 + "为什么" | 递进辩论（逐个补充） | 2-3 个 |
| 问题涉及多个技艺比较 | 并列辩论（同时发言→对比） | 2-3 个 |
| 问题涉及技艺传承/困境 | 多视角辩论（全面分析） | 全部 3 个 |
| 简单事实查询 | 不触发辩论，单 Agent 回答 | 1 个 |

### 2.3 辩论数据结构

```python
@dataclass
class DebateRound:
    round_num: int
    agent_name: str
    agent_avatar: str       # 头像/图标
    role: str                # "主答" | "补充" | "质疑" | "回应"
    content: str
    references: list[str]    # 引用的知识来源

@dataclass  
class DebateSession:
    question: str
    rounds: list[DebateRound]
    final_synthesis: str
    key_insights: list[str]  # 辩论中产生的关键洞见
```

## 3. 检索能力升级

### 3.1 现有问题

- 仅有关键词匹配检索（`retriever.py` 用 `in` 操作符）
- ChromaDB 依赖已声明但未实际使用
- 无向量语义检索
- 无混合检索融合

### 3.2 升级方案

```
用户问题
    │
    ▼
① Query Rewriting: "景泰蓝工艺失传原因" → ["景泰蓝 传承现状", "景泰蓝 工艺难度", "景泰蓝 传承人数量"]
    │
    ▼
② 多路检索:
   ├── BM25 关键词 (rank_bm25 + jieba 分词)
   ├── 向量语义 (ChromaDB + BGE Embedding)
   └── 图谱检索 (NetworkX 沿关系遍历)
    │
    ▼
③ RRF 融合 → Top-10
    │
    ▼
④ BGE-Reranker Cross-Encoder 精排 → Top-5
    │
    ▼
⑤ 送入 Agent 辩论 + 生成
```

### 3.3 新增模块

| 模块 | 文件 | 功能 |
|------|------|------|
| Embedding | `src/retrieval/embeddings.py` | BGE 模型管理 |
| BM25 | `src/retrieval/bm25_retriever.py` | 关键词检索 |
| Vector | `src/retrieval/vector_retriever.py` | ChromaDB 语义检索 |
| Fusion | `src/retrieval/fusion.py` | RRF 融合 |
| Reranker | `src/retrieval/reranker.py` | Cross-Encoder 精排 |
| Rewriter | `src/retrieval/query_rewriter.py` | 查询重写 |

## 4. 知识图谱可视化

### 4.1 交互式探索面板

```
┌─────────────────────────────────────────────┐
│  🕸️ 技艺关系图谱                             │
│  ┌─────────────────────────────────────┐    │
│  │                                     │    │
│  │    [紫砂泥]──uses_material──→[宜兴紫砂]  │
│  │       │                         │    │    │
│  │       │                    mastered_by │
│  │       │                         │    │    │
│  │    [宜兴]──originates_from──[陈师傅]    │    │
│  │                                     │    │
│  │  点击节点 → 展开详情面板               │    │
│  └─────────────────────────────────────┘    │
│                                             │
│  筛选: [全部] [材料] [工具] [传承人] [产地]   │
│  布局: [力导向] [层级] [环形]                │
└─────────────────────────────────────────────┘
```

### 4.2 实现方案

- 使用 **pyvis** 生成交互式图谱（支持节点点击、拖拽、缩放）
- Streamlit 中用 `st.components.v1.html` 嵌入
- 节点颜色按类型区分（技艺/材料/工具/传承人/产地/朝代）
- 点击节点 → 侧边弹出详情卡片

## 5. UI 整体升级

### 5.1 布局重设计

```
┌──────────────────────────┬──────────────────────────┐
│  💬 非遗问答对话区 (60%)   │  📊 知识增强面板 (40%)    │
│                          │                          │
│  用户: 景泰蓝为什么...     │  ┌ 切换视图 ────────────┐ │
│                          │  │ ○ 辩论时间线          │ │
│  🎨 张大师(景泰蓝):       │  │ ○ 知识图谱探索        │ │
│  "景泰蓝六道工序..."      │  │ ○ 引用溯源            │ │
│                          │  └──────────────────────┘ │
│  🏛️ 传承专家:             │                          │
│  "全国传承人不到50位..."   │  当前: 辩论时间线         │
│                          │  ┌──────────────────────┐ │
│  📜 历史专家:              │  │ Round 1 🎨 主答       │ │
│  "明清宫廷御用..."         │  │ Round 2 🏛️ 补充       │ │
│                          │  │ Round 3 📜 历史视角   │ │
│                          │  └──────────────────────┘ │
│  ─────────────────────   │                          │
│  🤖 综合回答:            │  关键洞见:               │
│  "景泰蓝的传承困境..."    │  • 工艺复杂度高          │
│                          │  • 传承人老龄化          │
│  [输入框________________] │  • 材料成本昂贵          │
└──────────────────────────┴──────────────────────────┘
```

### 5.2 视觉风格

- 配色：暖棕/古铜/宣纸白（非遗文化感）
- Agent 头像：每个 Agent 有独特图标和配色
- 辩论气泡：不同 Agent 用不同颜色/边框
- 时间线：左侧竖线 + 圆点，类似聊天记录但更有层次
